Water Mitigation Claims Management Platform

An end-to-end Django web application that manages the full lifecycle of water damage insurance claims — from property intake and document generation through Encircle integration, AI-powered field verification, and ALE lease management.
Django 4.2 PostgreSQL Celery + Redis Claude Vision AI Encircle API ReportLab / WeasyPrint Docker + Nginx Bootstrap 5

Project Overview

Built for a water mitigation and restoration company, this platform replaces a collection of disconnected spreadsheets, manual PDF workflows, and phone-based coordination with a single web application accessible from any device. It manages every stage of a mitigation claim — from the moment a property owner is entered to the final document handoff to insurers.

The system integrates with Encircle (an industry field documentation platform), uses Claude Vision AI for automated image analysis, generates and emails thermal-printer-ready labels, populates Excel estimate templates, tracks ALE leases through a multi-stage pipeline, and provides a full file server with folder-level versioning. Background tasks (Celery) handle all heavy operations asynchronously so the UI stays responsive.

AI-Powered Features (Claude Vision)

🤖 Sensor Image Renamer
  • Drag-and-drop upload of sensor display photos (MC meters and RH/T/GPP loggers)
  • Claude Vision reads the numeric values directly off each sensor screen
  • Renames every photo with its readings and sorts into 5 organized subfolders automatically
  • Parallel processing via ThreadPoolExecutor (4+ concurrent API calls)
  • Real-time progress bar via AJAX polling; manual correction panel for unreadable screens
  • Per-subfolder and full-session ZIP download
✅ Equipment Documentation Checker
  • Upload an Encircle room-photo PDF report and/or individual job site photos
  • Paste scope-of-work line items (Room | Description format)
  • Claude Vision cross-references photos against an equipment reference PDF (HUTCH EQPT LIST)
  • Returns FOUND / PARTIAL / NOT FOUND status with specific notes per line item
  • Filter by status, search by keyword, export results as CSV
  • Designed to catch documentation gaps before submitting to insurers

Notable Engineering Decisions

  • Fully asynchronous heavy operations — label generation, Encircle pushes, document population, and AI image analysis all run as Celery tasks; the browser polls for completion without blocking
  • Sub-claim auto-splitting — when 8000-series or 9000-series rooms are added during claim creation, the backend automatically creates separate MIT and RHT sub-claims with all insurance fields inherited from the primary claim
  • Reference-document-augmented AI — the equipment checker sends the company's own equipment photo guide (HUTCH EQPT LIST.pdf) as a Claude document block alongside job site photos, giving the model industry-specific context to make more accurate determinations
  • Session-based temp storage — AI tools use UUID-namespaced media directories so concurrent users never interfere with each other's in-progress sessions
  • Hash-based change detection — every file modification is fingerprinted; the audit log records who changed what, when, and from which IP address
  • Email open tracking — a 1×1 tracking pixel returns a redirect, logs the open event (IP, user-agent, timestamp), and can trigger admin notifications
  • Single Docker Compose stack — web, Celery worker (4 concurrency), Celery Beat, Redis, PostgreSQL, Nginx, and a headless LibreOffice UNO service all run in one compose file with shared named volumes
